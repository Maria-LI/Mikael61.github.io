(X)HTML-övning 2

Sektionering, element div och span

I dessa övningar skall du arbeta med att dela in ditt dokument i sektioner med avseende på grupper av element som har något gemensamt. I andra XML-tillämpningar, som t ex TEI (för digitalisering av tryckt material), finns element definierade som avser att inkludera grupper av element med likartad betydelse eller funktion. Ett exempel i TEI är fileDesc (för bibliografisk beskrivning) som innehåller titleStmt, publicationStmt och sourceDesc, vilka i sin tur innehåller uppgifter nödvändiga för att identifiera ett dokument, dess förlagor och omständigheter kring publiceringen.

Något motsvarande finns inte definierat i HTML, och om man önskar använda HTML för att strukturera en text på ett meningsfullt sätt och samtidigt göra det möjligt att med hjälp av CSS modifiera exempelvis placeringen av grupper av element (som en navigationsmeny) så kan man göra bruk av ett element div, som på förhand inte har någon som helst inneboende betydelse. Elementet div kan dock ges två användbara attribut id och class. 

<div class="kapitel">
  <h2> ... </h2>
  <p> ... </p>
  <p> ... </p>
  <p> ... </p>
</div>
<div class="kapitel">
  <h2> ... </h2>
  <p> ... </p>
  <p> ... </p>
  <p> ... </p>
</div>

kan exempelvis vara ett sätt på vilket det är möjligt att sektionera en text bestående av flera kapitel. Eftersom det inte finns något element för kapitelindelning i HTML så skapar man en klass av div-element, för att liksom emulera ett egenhändigt definierat kapitel-element.

Observera att värdet är helt och hållet godtyckligt men det är bra om det är mnemoniskt, dvs om det är ett uttryck som hjälper till att minnas för vilket ändamål det skapats. Det skall inte innehålla mellanslag, och för enkelhets skull endast bokstäverna a-z, A-Z, eventuellt följt av någon siffra.

Om det i stället är så att man har en typ av sektion som man vet med sig alltid endast förekommer en enda gång i ett dokument kan man med fördel använda attributet id. id kan, liksom class, åsättas i stort sett varje element. Skillnaden är att det i ett dokument inte får finnas två element med samma värde för id. 

<div id="leftMenu">
  <ul> 
    <li> ... </li>
    <li> ... </li>
    <li> ... </li>
    <li> ... </li>
  </ul>
</div>

är ett sätt att skapa en behållare för en navigationsmeny.

Elementet span kan användas på motsvarande sätt, men på "frasnivå", dvs skillnaden är att div är ett block-element och span ett inline-element. span kan därmed inte innehålla block-element (som h1 eller p). 

<span class="personNamn">Mikael Gunnarsson
</span>

är ett möjligt sätt att förfara med markeringen av personnamn i en text, om man finner ett behov av att kunna göra det.

Din uppgift är nu att ta fram någon tidigare version av Valans spådom och skapa sektioner i den, namngivna med class eller id. Se det som en förberedelse inför nästföljande övningar i CSS, där du bl a skall modifiera placeringen av dem. Du behöver givetvis inte skapa div-sektioner om där redan finns ett HTML-element som i stället kan ges class eller id. Exemplet ovan med navigationsmenyn, givet att den inte skall innehålla något annat en en onumrerad lista, är bättre att hantera enligt nedan. 

  <ul id="leftMenu"> 
    <li> ... </li>
    <li> ... </li>
    <li> ... </li>
    <li> ... </li>
  </ul>
